Some mathematical concepts that are helpful in 3D graphics.

Trigonometry

Matrices

A nice way to write equations that 'have something in common'.

For example, say that we want to rotate a point [x,y] through an angle.

x' = x * some_operation1(angle)
y' = y * some_operation2(angle)

This would be the rotation matrix.

Multiplication

If the target is row 2 and column 3:

  1. Take the first element from row 2 of A, multiply by the first element of column 3 of B.
  2. Take the second element from row 2 of A, multiply by the second element of column 3 of B.
  3. ...
  4. Sum it up

So we end up with:

x' = x*cos(angle) - y*sin(angle)
y' = x*sin(angle) + y*cos(angle)

Yes but why matrices?

A matrix can be used to describe rotation, translation and scaling. Matrices can be multiplied to represent a series of such transformations. Imagine if you have a turret on a tank. You can use one matrix to describe the rotation of the turret relative to the tank body. Another matrix to describe the rotation of the tank body relative to the ground and multiply them together to get the turret rotation relative to the ground.

In fact matrices are so useful matrix math is often implemented in hardware on GPUs.
